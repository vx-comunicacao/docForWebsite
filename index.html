Não estilize tags

Salvo em casos específicos, como na exibição do conteúdo textual do wordpress e o css reset

CSS reset mínimo

Evite usar de IDs

Possui pouco reaproveitamento

Nomenclatura de classes

BEM(Block Element Modifier)
Traz um padão ao código
Dimínui a colisão de classes

usaremos aqui com variação nos modificadores

Como exemplo vou usar um card

.card
.card__title
.card__thumb
.card__description
.card__buttom

Onde o card é o bloco e as classes card__xxx são os elementos que pertencem ao bloco.

Modificadores

Podem ser usados no bloco como nos elementos

Supondo por exemplo que em um outro local do site o mesmo card possua um fundo diferente.

Originalmente a metodologia BEM usaria um modificadores dessa maneira

.card.card__fundo-diferente

Em outro lugar com uma borda superior e o fundo diferente, neste caso são dois modificadores.

.card.card__fundo-diferente.card__com-borda-superior

Observe como a declaração css começa a ficar grande e repetitiva.

Agora reescrevendo com modificadores is- e has- que são encontrados em outras metodologias e no framework bulma, teremos menos repetição e apenas olhando o código poderemos identificar o modificador.

.card.has-fundo-diferente.has-com-borda-superior
.card.is-open

No bootstrap temos a classe btn e a classe panel, as duas possuem uma variação para a estilização de cor primary que modifica a cor padrão.

No botão
button class=”btn btn-primary"

No panel
class=”panel panel-primary”

por que não é simplesmente

btn is-primary
panel is-primary

CSS funcional/Utility-first como modificadores

Deve ser imutável, não pode mudar ao longo da execução da página, exceto no uso de beakpoint.

1 - Css funcional
Basicamente traz para a classe uma propriedade:valor do css
.inline-block esta classe representa a propriedade/valor display:inline-block
.inline esta classe representa a propriedade/valor display:inline
.inline-block esta classe representa a propriedade/valor display:block

2 - Utilitários
Muito parecidos com o css funcional mas suas propriedades/valores possuem características que podem mudar em cada projeto, com valores provenientes do design.

Os exemplos mais comuns são cor da fonte, cor do fundo, tamanho da fonte, margins e paddings.
color = cor da fonte

.color-primary = color:#ff1212
.color-brand = color:#ff1212
bg = cor da fonte

.bg-primary = background-color:#ff1212
.bg-brand = background-color:#ff1212

supondo que se tenha o componente card que por padrão possua um título que é preto, mas em algum outro local do site esta cor é azul( definida como secondary ), pode não ser muito produtivo ou reaproveitável utilizar um modificador do BEM

h2 class=”card__title is-nome-do-modificador ”

Com certeza é mais produtivo usar desta maneira
h2 class=”card__title color-secondary”

ou uma fonte maior

h2 class=”card__title fs-large”

Lembrando que é necessário que os valores seja definidos no início do projeto de acordo com as especificações do design.

Um bom exemplo para uso desta técnica é a criação de títulos do site, apenas adicionando algumas classes.

h1 class=”color-primary fs-medium ff-lato fw-bold”
Fast and Simple

Em alguns casos o uso do css funcional e os utilitários pode substituir a criação de classes css, mas isto tem efeitos colaterais como dificuldade de manutenção, dificuldade em ajustes finos, refatoração/redesign e manutenção, portanto use com moderação e sabedoria.
